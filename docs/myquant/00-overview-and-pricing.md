# 00 概览 & 版本/付费分层

## 掘金量化的 SDK 与数据分层

掘金量化（myQuant）按用户身份分为：

| 版本 | 调用方式 | 数据/接口范围 |
|---|---|---|
| **体验版（免费）** | 注册即用，本地终端 | 公版数据接口全部可调；K 线日/分钟、财报、估值/市值、指数成分股、交易日历、标的池等 |
| **专业版（付费个人）** | 个人付费订阅 | 同公版 + 增值数据（行业/板块/分红/股东/龙虎榜/北向/资金流/L2 部分） |
| **券商版** | 与合作券商绑定 | 公版接口在券商版会以「升级提示」形式存在；L2 等高频数据由具体券商决定 |
| **机构版** | 机构客户 | 全量 + 私有部署 |

> 调用「公版（体验版/专业版/机构版）」函数在文档原文中明确写为：
> > 「此函数为掘金公版(体验版/专业版/机构版)函数，券商版以升级提示为准」
>
> 因此**默认免费可用**的 API 在文档中以这条提示打标。

## 「免费 / 付费」的判定规则

文档站直接通过**侧栏分组名**进行区分（左侧 `API介绍` 折叠组）：

### 免费组（页头明确写「（免费）」）

- **行情数据查询函数（免费）** — `last_tick` / `current_price` / `history` / `history_n` / `context.data`
- **通用数据函数（免费）** — `get_symbol_infos` / `get_symbols` / `get_history_symbol` / 交易日历 / 交易时段 / 合约到期
- **股票财务数据及基础数据函数（免费）** — `stk_get_fundamentals_*` / `stk_get_finance_prime/deriv` / `stk_get_daily_valuation/mktvalue/basic` / `stk_get_index_constituents`（含截面 `_pt` 变体）
- **期货基础数据函数（免费）**（本目录不展开）

### 付费组（页头明确写「（付费）」）

- **股票增值数据函数（付费）** — 22 个 `stk_get_*` 函数，覆盖行业、板块、分红、股东、龙虎榜、北向、资金流、业绩预告等
- **期货增值数据函数（付费）** / **基金增值数据函数（付费）** / **可转债增值数据函数（付费）**（本目录不展开）

### 例外：免费页里的付费函数

「行情数据查询函数（免费）」页里有 **5 个 L2 函数** 单独注明：

> 「**仅特定券商付费提供**」

它们是：

- `get_history_l2ticks`
- `get_history_l2bars`
- `get_history_l2transactions`
- `get_history_l2orders`（只支持深市）
- `get_history_l2orders_queue`

### 交易/事件类（不分免费付费）

- 基本函数 / 数据订阅 / 数据事件 / 交易函数 / 交易查询函数 / 两融 / 算法 / 新股新债 / 基金交易 / 债券交易 / 交易事件 / 动态参数 / 标的池 / 其他函数 / 其他事件

这些是「策略执行层」而非「数据获取层」，**功能本身免费**，但能否真实下单/查询持仓取决于绑定的交易账户权限（实盘账户 + 券商通道）。

## 数据日清洗时点（容易踩坑）

> 「**盘后 18 点清洗入库更新当天数据**，需要盘后取数据的，请在 18 点后取。」
>
> 来源：`history` / `history_n` 函数的「注意」第 8 条。

意味着：
- 14:59:00 调 `history` 取「2026-06-09」日线，可能拿不到当日 K 线
- 18:00 之后再调，当日数据才稳定
- 交易日 20:00 才会更新当日的龙虎榜、北向资金等增值数据

---

## 单次返回数据量上限

- `history` / `history_n` 单次最多返回 **33000 条**，超出部分不返回
- `get_history_l2ticks` 每次只能取 **一天**；超过 31 天直接拿不到
- `get_history_l2bars` 每次最多 **31 天**
- `get_history_l2transactions` / `get_history_l2orders` / `get_history_l2orders_queue` 每次只能取 **一天**
